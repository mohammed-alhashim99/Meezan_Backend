## Live API

**Base URL:** `https://meezan-backend-v76h.onrender.com/api`

> ⚠️ Free tier — the backend sleeps after inactivity. The first request may take up to **60 seconds** to wake up. Visit the health endpoint first.

**Health check:** `GET /api/health/` → `{"status": "ok", "message": "Meezan API is running"}`

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/health/` | Service health check |
| `POST` | `/api/upload/` | Parse a bank statement (CSV or PDF) |
| `POST` | `/api/categorize/` | AI-categorise transactions (Gemini) |
| `POST` | `/api/insights/` | Generate AI savings insights (Gemini) |

### POST /api/upload/
Upload a CSV or PDF bank statement.

**Request:** `multipart/form-data` with field `file`

**Response:**
```json
{
  "transactions": [
    { "date": "2026-04-01", "description": "راتب شهر أبريل", "amount": 12000.0, "balance": 12000.0 }
  ],
  "count": 30,
  "currency": "SAR"
}
```

### POST /api/categorize/
Categorise a list of transactions using Google Gemini AI.

**Request body:**
```json
{ "transactions": [ { "description": "ستاربكس العليا", "amount": -48.0 } ] }
```

**Response:** Same transactions with an added `"category"` field.

**8 categories:** Food & Dining, Transport, Shopping, Bills & Utilities, Health, Entertainment, Transfers, Other

### POST /api/insights/
Generate personalised savings insights from categorised transactions.

**Request body:**
```json
{ "transactions": [ { "description": "...", "amount": -48.0, "category": "Food & Dining" } ] }
```

**Response:**
```json
{
  "insights": [
    { "title": "إنفاق المطاعم مرتفع", "description": "...", "type": "warning", "saving": 200.0 }
  ],
  "total_spent": 5885.0,
  "top_category": "Food & Dining"
}
```

## Local Development

### 1. Clone & set up virtualenv

```bash
git clone https://github.com/mohammed-alhashim99/Meezan_Backend.git
cd Meezan_Backend

python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Create `.env`

Copy the example and fill in your values:

```bash
cp .env.example .env
```

`.env` contents:
```
SECRET_KEY=your-secret-key-here
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1
CORS_ALLOWED_ORIGINS=http://localhost:5173
GEMINI_API_KEY=your-gemini-api-key
```

Get a free Gemini API key at [aistudio.google.com](https://aistudio.google.com/app/apikey)

### 3. Run migrations & start server

```bash
python manage.py migrate
python manage.py runserver
```

API is now available at `http://localhost:8000/api/`

## Deployment (Render)

The repo includes `render.yaml` for one-click deployment on [Render](https://render.com).

**Required environment variables on Render:**
| Variable | Value |
|----------|-------|
| `DEBUG` | `False` |
| `SECRET_KEY` | auto-generated by Render |
| `ALLOWED_HOSTS` | `meezan-backend-v76h.onrender.com` |
| `CORS_ALLOWED_ORIGINS` | `https://your-frontend.vercel.app` |
| `GEMINI_API_KEY` | your Gemini API key |

> `ALLOWED_HOSTS` wildcards for `.onrender.com` and `.vercel.app` are set directly in `settings.py` so the service works even without the env var.

## Scope & Constraints

This project was built under a 14-day contract. The following are intentionally **out of scope**:

- ❌ No user authentication / JWT
- ❌ No database persistence (stateless API)
- ❌ No bank API integrations
- ✅ Stateless: every request is self-contained
